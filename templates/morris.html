Introduction

When it comes to dealing with larger operations, learning how to connect remotely to servers is important. Secure Shell (SSH) gives you the ability to connect securely over unsecure networks. Knowing how to navigate this tool will aide you massively in your journey.

Applications

Terminals are typically used for these connections, though some applications (I.E. Text Editors/IDEs) have built-in functionality. Every operating system comes with a terminal, and at times the default will serve you well out of the box (see the macOS Terminal).

Here's a list of a few popular applications.

  • PuTTY (Linux/Mac/Windows)
  • Cyberduck (Mac/Windows)
  • Terminal.app (Mac)
  • OpenSSH (Unix)
  • Powershell (Windows)
  • WinSCP (Windows)
  • Tera Term Pro (Windows)
    • Your First Connection

    What throws a lot of people off initially is how the connection command is to be entered. Simply type in the following.

    ssh user@ipaddress

    In practice your actual input should look more like the following:

    ssh admin@192.168.1.1

    After hitting enter/return and a connection is made successfully you will be prompted for the password and it should appear below the previous command you just sent.

    admin@192.168.1.1's password:â–ˆ

    If the password is correct, you'll back taken to the root directory of your server. If it's incorrect, you'll either be prompted again or will need to resend the ssh login info, including the IP.

    Commands To Avoid

    These commands will remove directories and/or files. Use them with caution.

    rm -rf /path/
    rm -r /path/
    rm -f /path/
    Cron

    Accessing the server allows you to directory work with the Cron itself. It is a time based job scheduler that administrators use to execute jobs automatically at set intervals. Updating memberships daily to account for rollovers (current to grace and grace to expired) for example. One useful command I've found personally helpful for checking and locating the Cron is:

    ps -ef | grep cron | grep -v grep

    Learning to properly locate the Cron and create Crontabs is a valuable skill. Don't be afraid to Google more about creating Crontabs.
